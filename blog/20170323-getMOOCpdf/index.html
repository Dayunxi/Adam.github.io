制作一个爬取中国大学MOOC课件的爬虫

发表于 2017-03-23 | 分类于 技术 | | 阅读次数

说明

首先MOOC中的 robots协议 并没有禁止爬虫爬取
以Requests, Re库为主要技术路线
MOOC并不是静态页面，而是ajax异步加载的，只能通过分析post请求去想办法获取文件

背景

有的时候想把某一门课的PPT下下来，但是MOOC却并没有提供一个全部下载的接口，而自己一个一个下载实在太麻烦，因此打算做一个自动爬取所有课件的爬虫。爬取视频也是类似的，不过虽然没明确说禁止，但个人感觉爬取视频可能有点不太好。。。

设计

需要三个输入参数：学校ID，课程ID(tid)，保存路径
以“Python网络爬虫与信息提取”这门课为例，课程地址如下
http://www.icourse163.org/learn/BIT-1001870001?tid=1001962001#/learn/content
其中“BIT-1001870001”代表schoolId，“1001962001”代表termId
只需要将这两个字符串作为参数输入程序，即可获取所有的课件的pdf格式下载

PS. 感觉自己的用户体验设计的不错，嘻嘻

实现过程

分析POST请求

首先分析文档的下载链接是怎么来的，在浏览器开发人员工具下发现GET链接

http://www.icourse163.org/learn/BIT-1001870001?tid=1001962001

得到的Document只有一个外层框架，并没有实际内容，因此就知道MOOC是异步加载的页面，不能通过普通的GET请求获取下载链接
然后点进任意一个文档查看页面，大概看了下所有的请求的响应值，然而没看出下载链接是在哪个请求。然后打开Fiddler，在视频与文档间切换界面时，通过Fiddler发现产生了几个POST请求，其中一个

http://www.icourse163.org/dwr/call/plaincall/CourseBean.getLessonUnitLearnVo.dwr

进去后终于在里面发现了一个下载pdf的链接地址。

然后开始研究post的参数格式（Python代码中有其具体格式形式），发现需要httpSessionId和另几个类似于文档id的参数，还有一个batchId。
batchId由一串数字组成，根据经验判断，看着很像unix时间戳，然后边查看了一下当前的unix时间戳，发现前面几位都是一致的，但是batchId多了3位，猜测是以毫秒为单位的时间戳。

接着返回上一级界面（课程目录），在F12的网络中找到.dwr结尾的XHR文件，其中一个的响应文本比较大，有点可疑，拷贝至notepad查找之前看到的类似于id的参数，果然找到了，于是确定

http://www.icourse163.org/dwr/call/plaincall/CourseBean.getMocTermDto.dwr

是我们要找的POST请求链接，通过分析发现该请求返回了所有课件的信息（包括PPT,视频，富文本等），有几百行，每一行大概如下

s12.anchorQuestions=null;s12.attachments=null;s12.chapterId=1002293023; s12.contentId=1004602244;s12.contentType=3;s12.gmtCreate=null;s12.gmtModified=null; s12.id=1003046433;s12.jsonContent=null;s12.lessonId=1002627077;s12.name=”\u5168 \u8BFE\u7A0B\u5185\u5BB9\u5BFC\u5B66\uFF08WS00\u5355\u5143\uFF09\u5B66\u4E60\u8D44\u6599”; s12.position=null;s12.resourceInfo=null;s12.termId=1001962001;s12.viewStatus=null;

然后发现所有的文档都有一个共同特征，即他们的contentType均为3，然后在里面提取出了四个重要信息：contentType, contentId, id, name
与之前的几个参数是对应的。而该post请求只需要termId(tid)。

POST请求结构

总结

至此所有思路就理清楚了：

  1. 构造headers (需要tid，schoolId, httpSessionId)
  2. 构造POST(CourseBean.getMocTermDto.dwr)所需的参数 (需要tid, httpSessionId)
  3. 通过requests.post()方法获取响应数据
  4. 通过RE库正则匹配所有课件的信息(id, contentId, name) (contentType = 3)
  5. 通过循环使用requests.post()从CourseBean.getLessonUnitLearnVo.dwr获取下载链接，并将文件以name命名下载至指定目录

本来还想爬视频，但觉得爬取视频可能有点不太好，就算了。但过程是类似的，将contentType改成1，正则匹配稍微改下就能获取视频源的信息
然后在响应内容里可以看到

  • flvHdUrl
  • flvSdUrl
  • flvShdUrl
  • mp4HdUrl
  • mp4SdUrl
  • mp4ShdUrl

六种格式的下载链接，requests.get()保存下来即可（不过不建议这样做）。
